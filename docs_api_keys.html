API Keys Management

To interact with the Alconio API or send data from your SDKs, you'll need at least one valid API Key. These keys act as both a unique identifier and a security credential for your property.

Key Types

  • Publishable Key: Safe to include in client-side code (frontend JavaScript). It only allows data ingestion and cannot be used to read your analytics or change settings.
  • Secret Key: Used for server-to-server communication and administrative tasks. Never expose this key in your frontend or public repositories.

Generating Keys

Navigate to Settings > API Keys to see your existing keys. You can create multiple keys for different environments (e.g., Development, Staging, Production). When you generate a new Secret Key, Alconio will only show it to you once. Make sure to copy it to a secure location immediately.

Key Rotation & Expiry

As a security best practice, we recommend rotating your API keys every 90 days. To rotate a key without downtime:

  1. Generate a new key in the dashboard.
  2. Update your application to use the new key.
  3. Wait a few hours to ensure all traffic has transitioned.
  4. Deactivate and delete the old key from the dashboard.

Security Basics

If you suspect an API key has been compromised, you should revoke it immediately. Alconio also supports Domain Whitelisting for Publishable Keys; you can specify which domains are allowed to send data using that key, preventing "referral spam" and unauthorized usage on other sites.
