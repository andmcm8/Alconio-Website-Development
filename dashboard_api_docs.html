Authentication

All API requests must include your Bearer token in the Authorization header. You can manage and rotate your API keys via the Settings > Developer tab.

// Standard Auth Header
Authorization: Bearer alc_live_92kd81...

SDK Installation

Integrate Alconio directly into your application using our official SDKs for Node.js, Python, and Go.

JavaScript / Node.js

# Install via NPM
npm install @alconio/sdk

// Initialize client
const alconio = new Alconio('API_KEY');

Deno / JSR

# Direct import
import { Alconio } from "jsr:@alconio/sdk";

const client = new Alconio({ key: "..." });

Rate Limiting

We enforce rate limits to ensure stability across our platform. Limits are applied based on your subscription tier.

Tier Limit (RPM) Burst Allowance
Starter 1,000 50 requests
Business 10,000 500 requests
Enterprise Unlimited* Variable
GET

/v1/analytics/metrics

Retrieve aggregated metric summaries for a specific property and timeframe.

Parameters

Name Type Description
property_id string Required. Unique ID of the asset.
period enum Timeframe: "24h", "7d", "30d"

Curl Example

curl -G "https://api.alconio.com/v1/metrics" \
-d property_id=site_9210 \
-H "Authorization: Bearer YOUR_TOKEN"

Response

{ "status": "success", "data": { "total_users": 14209, "bounce_rate": "32.4%", "avg_session": "4m 12s", "active_now": 482 } }
GET

/v1/analytics/sessions

List recent user sessions with source, geolocation, and device metadata.

Parameters

Name Type Description
limit number Max records (1-100, default 20)
offset number Pagination offset
DELETE

/v1/data/purge

Irrevocably delete historical log data for the specified property. Use with caution.

Warning: Once purged, data cannot be recovered even by Alconio support.
GET

/v1/users

Fetch a list of distinct users tracked across your properties, including their last seen status and custom attributes.

Input Filters

segment "new" | "returning"
query Search by ID/Email

Response Body

{ "users": [ { "id": "u_812", "last_active": "2m ago" }, { "id": "u_901", "last_active": "1h ago" } ] }
GET

/v1/properties

Retrieve a collection of all digital properties (websites, apps) under your account.

POST

/v1/properties

Register a new property for monitoring. Requires an administrative API key.

// Request Payload
{
  "name": "Production Mobile App",
  "domain": "app.example.com",
  "type": "ios_app"
}
POST

/v1/events/track

Track custom user interactions or system triggers for processing in real-time.

// Payload Structure
{
  "event": "purchase_completed",
  "value": 99.00,
  "properties": {
    "curreny": "USD",
    "coupon": "SUMMER24"
  }
}

Webhooks

Receive real-time notifications about important events in your Alconio account via HTTP POST callbacks.

Supported Event Types

site.down

Fired when a monitored site becomes unreachable.

site.up

Fired when a site recovers from an outage.

metric.threshold

Fired when a performance metric exceeds targets.

report.ready

Fired when a monthly analytics report is generated.

Verification

All webhook requests include an X-Alconio-Signature header. Use your webhook secret to compute an HMAC-SHA256 hash of the JSON payload to verify the sender.

Error Reference

Alconio uses standard HTTP response codes to indicate the success or failure of an API request.

Code Definition Recommended Action
200 / 201 OK / Created Request successful.
400 Bad Request Check required parameters and JSON syntax.
401 Unauthorized Invalid or expired API token.
429 Rate Limit Exceeded Slow down or upgrade your plan.
500 Internal Server Error Contact Alconio support.
